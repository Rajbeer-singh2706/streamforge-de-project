Key learnings & principles

Idempotency by design: SHA-256 over (subscription_id + event_type + date) enables restart-safe deduplication without external state
Non-blocking Kafka discipline: poll(0) for delivery callbacks + flush() only on clean shutdown is a deliberate correctness choice, not a default
Smoke tests should accumulate failures: Running all 8 checks rather than failing fast surfaces the full health picture in one pass
Consumer group isolation per run: Timestamp-suffixed group IDs ensure smoke test consumers always read from offset 0 independently
